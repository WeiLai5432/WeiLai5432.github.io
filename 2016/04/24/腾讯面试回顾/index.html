腾讯面试回顾

发表于 2016-04-24 | 分类于 日常 | | 阅读次数

前几天做了阿里的笔试题，被虐哭了。开始说时间是四十分钟，二十个选择题。做完了给了八十分钟的附加题。三道大题，第一题是在 SQL 语句里查找相关关键词的问题，第二题直接看不懂。第三题是设计一个数据结构。跪惨了。

更一下鹅厂面试被虐的经历。

周日早上五点起来去火车站。早上车少开得快，不到六点就到了。七点十分的火车。上了火车睡一会觉，看了看 C++ 和数据结构，中午十二点就到了。北京的天气很好。下车坐地铁到了面试酒店附近，还有一段时间，又没地方去，就随便走走了。

先说面试。

从 THU 坐公交到面试的酒店。通知我四点半面试，不到四点就到了。坐了一小会就听见念我的名字，然后就拿着号牌去面试。

面试官挺年轻的，感觉人很 nice。问了我大概这么几个问题：（有些内容不记得了，顺序也忘了）

第一个问题是，从底层实现角度来解释一下函数的调用。

这个问题应该是很基础的，但是无奈自己太渣不太清楚，隐约记得哪看过或者讲过，没办法。然后我就乱说一顿，面试官还提示了一下我，从汇编语言的角度解释。不懂就是没办法啦。

简单地讲，函数（如strlen）的名称类似变量名称，指向内存中某个地址。区别在于变量指向的目标是某个
值，函数名指向的目标是某段代码。函数被调用前，调用者（如main）将当前代码执行位置压栈，然后跳向
内存中函数代码位置，，执行之后返回堆栈中保存的调用者（main）的原执行位置，弹栈后继续执行。
另，用户函数和系统函数在编译时一般是分开存放的。

更详细的见：http://blog.chinaunix.net/uid-23069658-id-3981406.html

了解内存对齐吗？

然后他看了看我的简历，问了我一个经典的问题，链表和数组的区别，优缺点。

我大概说了一下，当我说到在内存中连续存放时，他问我这有什么好处，我说寻址和访问速度快，然后他引导性地问我，如果一个占用空间为4字节的参数，在内存中存放的位置不是4的整数倍，那么怎么办。这其实就是内存对齐的内容，面试官还提示我可以增加内存，然后引导性地问了我几个问题，比如 CPU 如何读写内存，访问某个地方需要几次等。最后告诉我这就是内存对齐。

物理内存和虚拟内存
这个问题也是很经典的，不过我以前认为的虚拟内存的概念是不准确的。http://blog.chinaunix.net/uid-26983585-id-3364091.html

CPU 如何访问内存？

这个问题记不清了，大概是一个很大的字符串数组（内存能存下），统计26个字母出现的次数
后来上网搜了下，也是很经典的问题，就是先建一个26的数组，然后算字符串里的每个字符与a或A的差来确定下标。当时我提到了 map，好像还提到了hash。
中间我说好像 map 是哈希实现的，然后他比较惊讶问我那 STL 里面还有个 hash_map 是怎么实现的，我就懵了。其实 map 是红黑树实现的，效率是O(logn)

如何评价一个哈希算法的好坏？
我说了几个，比如哈希之后占用内存的大小，冲突的个数，哈希函数是否高效，其实还是挺接近的。还有能否将关键字均匀影射到哈希空间上，有无好的解决冲突的办法。

如果发生了冲突怎么办？
我说了数组里的移位，他引导我说如果用链表呢，其实就是链接到同一个链表里。

哈希的时间复杂度？
这个问题我都跪了，我一直想哈希函数去了。

了解哪些寻路算法？
回答了 Djikstra，还有 A＊，但是我把 A＊ 和几率型算法搞混了，一不小心说漏嘴说出个随机数，然后赶紧改口说估价函数。他问我如何设置估价函数能确保一定能找到最优解？这个真心不懂。http://blog.itpub.net/490225/viewspace-832739/

A*（A-Star)算法是一种静态路网中求解最短路最有效的方法。
公式表示为： f(n)=g(n)+h(n),
其中f(n) 是节点n从初始点到目标点的估价函数，
g(n) 是在状态空间中从初始节点到n节点的实际代价，
h(n)是从n到目标节点最佳路径的估计代价。
保证找到最短路径（最优解的）条件，关键在于估价函数h(n)的选取：
估价值h(n)<= n到目标节点的距离实际值，这种情况下，搜索的点数多，搜索范围大，效率低。但能得到最优解。
如果 估价值>实际值, 搜索的点数少，搜索范围小，效率高，但不能保证得到最优解。
估价值与实际值越接近，估价函数取得就越好。

这个时候面试已经接近尾声了，我自己都坐不住了。对我这样的渣渣，面试官已经足够有耐心了，然后又问了我几个象征性的问题。

你最近做的最有成就感的事是什么？（技术方面的）
我大概想了想有成就感的就是题目 AC 了吧但是又好久没刷题，没想起来最近做过啥难题。比赛成绩也不太拿得出手，就说了刚做的那个 iOS 项目。然后他问我这个项目的技术难点是什么？我就懵了，遇到的难点太多了但是都是因为自己渣，于是就说具体实现的时候卡了一些问题。

最近看过什么技术书籍？
我回答刚开始看《机器学习》，然后他有点疑惑地问我为什么要看机器学习？我说了解了一下之后比较感兴趣就开始看了。他问我你对自己的职业生涯规划是什么？显然我对自己并没有清晰的职业规划，就说目前是啥都想学，以后可能做算法或者相关的方向吧。

你还有什么问题？
我问他我啥时候能收到通知，他说这两天。我又问您觉得我有哪些需要提高的地方？他说基础知识应该夯实。可能是因为我的这个问题，他想再给我一次机会，就又问我笔试题怎么做的，我开始有点不明白，就说在宿舍做的啊，他说是自己做的吧？我说是啊。他问笔试题里有一道关于死锁的问题，里面提到了银行家算法，你说一下这个银行家算法是怎样的。然后我就懵了说不了解。他说你笔试遇到了问题也没有去查吗？这个问题我浑身冷汗，基本算是宣判了结果了。我只好说笔试涉及的问题比较多没有太多时间，当然了这个理由很苍白。

出了酒店感觉非常失落。感谢面试官的耐心，人真的很 nice。我也认识到自己有夺渣，明显欠学习。

晚上在腾讯视频大楼，就是播 NBA 的地方。我误以为季后赛还没开打，就没想着去找解说合影啥的。

抽空查了下微信的面试状态，毫无意外地跪了。
