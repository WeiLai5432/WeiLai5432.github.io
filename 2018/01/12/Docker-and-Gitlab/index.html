Docker and Gitlab

发表于 2018-01-12 | | 阅读次数

最近配置了 Docker 和 Gitlab，记录一下相关内容。

安装和基本设置 follow official documents。

Docker 常用命令为：

gitlab 配置文件位置在 /etc/gitlab/gitlab.rc

external_url #这个是 gitlab 页面上展示的 url，包括仓库的地址和一些帮助文档的显示，如果不是默认端口访问 gitlab，最好把端口号也加在这里

我们的设置是，docker 把外部的7000端口映射到 gitlab 容器的80端口，一开始没有写端口号，导致页面上的仓库IP没有端口号。
然后查找设置方法，有两个思路：

  1. 将 gitlab 容器端口由80改为7000，然后重新配置 docker 容器，将裸机的 7000 端口映射到容器同样的 7000 端口。
  2. 不修改端口设置，只在容器内的 gitlab 设置中把url（external_url）加上端口号，但要注意，裸机7000端口映射的容器端口仍然是80，因此 gitlab 在容器内仍然要监听80端口而不是external_url 的7000端口，所以要修改 nginx['listen_port'] = 80。
